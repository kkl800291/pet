# Pet Loop PRD — 产品需求文档

> 版本 v0.1 · 2026-03-23 · 面向 UI 重设计使用

---

## 一、产品定位

**Pet Loop** 是一款以宠物为核心实体的宠物社区产品。用户通过「宠物主人」身份注册，为自己的宠物创建档案，再以宠物视角发布日记、互动、私信。

目标用户：养宠物的年轻用户（18–35 岁），主要使用微信小程序。

## 三、核心数据模型

### 主人（Owner）
| 字段 | 说明 |
|------|------|
| name | 显示名 |
| phone | 手机号（登录凭证） |
| avatarUrl | 头像 |
| bio | 简介 |
| status | active / banned |

### 宠物（Pet）
| 字段 | 说明 |
|------|------|
| name | 宠物名 |
| species | 物种（猫 / 狗 / 其他） |
| breed | 品种 |
| gender | 性别 |
| birthDate | 出生日期 |
| avatarUrl | 宠物头像 |
| bio | 宠物简介 |
| visibility | public / followers / private |
| ownerId | 所属主人 |

### 帖子（Post）
| 字段 | 说明 |
|------|------|
| title | 标题 |
| content | 正文 |
| images | 图片列表（最多 9 张） |
| visibility | public / followers / private |
| status | pending（待审）/ approved（通过）/ rejected（驳回） |
| petId | 关联宠物 |
| ownerId | 发帖主人 |

### 关注（Follow）
- 主人关注主人（非宠物关注宠物）
- 互相关注后可私信（默认规则）

### 私信（Message / Thread）
- 每对主人之间一个 Thread
- Thread 包含多条 Message

### 举报（Report）
| 字段 | 说明 |
|------|------|
| targetType | post / pet / owner / comment |
| targetId | 被举报对象 ID |
| reason | 举报原因（文字） |
| status | open / resolved |

## 四、用户端（小程序）功能模块

### 4.1 认证
- **手机号登录**：微信手机号授权一键登录
- 首次登录引导创建「主人档案」+ 至少一只宠物

### 4.2 首页 Feed
- 展示已通过审核的公开帖子（瀑布流）
- 按时间倒序，支持下拉刷新 / 上拉加载更多
- 每张卡片展示：宠物头像、宠物名、帖子图片、标题、点赞数、评论数
- 顶部 Tab：**推荐** / **关注**

### 4.3 帖子详情
- 图片轮播（横划）
- 正文、标签
- 点赞 / 取消点赞
- 评论列表 + 发评论
- 举报入口

### 4.4 发帖
- 选择关联宠物（下拉）
- 上传图片（最多 9 张，支持相册 + 拍照）
- 填写标题 + 正文
- 选择可见性（公开 / 仅粉丝 / 仅自己）
- 提交后进入审核队列（状态：待审）

### 4.5 宠物档案页
- 宠物基本信息 + 封面图
- 该宠物的历史帖子（网格排列）
- 主人信息入口
- 关注 / 取消关注主人

### 4.6 主人主页
- 主人头像、名字、简介
- 养了多少只宠物、粉丝数、关注数
- 宠物列表（横向滚动卡片）
- 最近帖子

### 4.7 消息 / 私信
- 会话列表（按最新消息排序）
- 会话详情：气泡式聊天，WebSocket 实时推送
- 入口条件：互相关注后可发起私信（默认规则，可后台调整）

### 4.8 我的
- 编辑主人档案（头像、昵称、简介）
- 我的宠物列表（添加 / 编辑宠物）
- 我的帖子（全部 / 按状态）
- 设置（退出登录）

## 五、管理后台功能模块

### 5.1 概览（Dashboard）
- 数据看板：用户数、宠物数、待审内容数、待处理举报数、审计记录数
- 快速入口：去审核、去处理举报

### 5.2 内容审核
- 列表（待审 / 已通过 / 已驳回）+ 搜索
- 左列表右详情：查看帖子图片、正文、宠物、主人信息
- 操作：通过 / 驳回

### 5.3 举报处理
- 列表（待处理 / 已处理）+ 搜索
- 查看举报详情：目标类型、目标内容、举报人、举报原因
- 操作：标记已处理

### 5.4 用户管理
- 用户列表（搜索姓名 / 手机号）
- 查看用户状态
- 操作：封禁 / 解封

### 5.5 宠物管理
- 宠物列表（搜索宠物名 / 品种）
- 查看可见性、所属主人
- 只读，不直接操作（举报流程处理）

### 5.6 风控记录（审计日志）
- 记录所有管理员操作：通过/驳回帖子、封禁/解封用户、处理举报、修改配置
- 字段：动作、目标类型、执行人、时间、详情

### 5.7 基础配置
| 配置项 | 可选值 |
|--------|--------|
| 默认帖子可见性 | 公开 / 粉丝可见 / 仅自己 |
| 私聊规则 | 互相关注后可聊 / 评论后可聊 / 开放私聊 |
| 审核模式 | 机审 + 人审 / 纯人工 / 先发后审 |
| 评论审核 | 开启 / 关闭 |
| 审计保留天数 | 数字（默认 90） |

---

## 六、内容审核规则

- **机审关键词**：加微信、兼职、广告、引流、投资、返利、代购、博彩、贷款
- 命中关键词 → 状态自动置为 `pending`，进入人审队列
- 未命中 → 根据「审核模式」决定是否直接通过

## 八、设计规范建议

### 品牌基调
- 温暖、轻松、治愈感
- 以「宠物」作为视觉主角，主人是辅助角色

### 色彩
- 主色：暖橙 `#FF7043` 或活力蓝 `#2563EB`（二选一定调）
- 背景：米白 `#FAFAF8`，卡片白 `#FFFFFF`
- 文字：深灰 `#1A1A2E`，辅助灰 `#6B7280`

### 组件规范
- 圆角统一：卡片 16px，按钮 12px，头像 圆形
- 宠物卡片：头像 + 名字 + 品种，突出宠物图片
- 帖子卡片：图片优先（方形裁切），文字次之
- 底部导航：首页 / 发现 / 发帖（+号）/ 消息 / 我的

### 交互
- 点赞：动画反馈（心形弹跳）
- 图片上传：压缩至 1MB 以内再上传
- 状态反馈：审核中帖子显示「审核中」角标

## 九、当前已知技术约束

| 项目 | 说明 |
|------|------|
| 后端 API | `http://127.0.0.1:4000`，RESTful JSON |
